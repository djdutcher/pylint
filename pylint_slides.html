Pylint

Code Analysis for Python

Uses

  • Error detection
  • Validating coding standards

Install

sudo pip install pylint

Run

pylint -E example.py

Reported Messages

Convention For standards violations
Refactor For bad code smells
Warning For possible bugs
Error For bugs

Convention Messages

  • invalid-name
  • bad-whitespace
  • missing-final-newline

Refactor Messages

  • too-many-locals
  • too-many-branches
  • too-many-statements

Warning Messages

  • unused-argument
  • unused-variable
  • broad-except

Describing Error Messages

pylint --help-msg <msg-name>

pylint --help-msg broad-except

:broad-except (W0703): *Catching too general exception %s*
  Used when an except catches a too general exception, possibly burying
  unrelated errors. This message belongs to the exceptions checker.

Customizing Validation Rules

Rules are controlled by a config file

Pylint can generate a default config file

pylint --generate-rcfile > .pylintrc

Configuration File

Reg Exs for valid class / function / variable names

Enable / disable style rules

.pylintrc in your home folder will be used automatically
